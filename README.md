## Tool switcher для ЧПУ

Небольшое десктоп‑приложение на Java, которое **автоматизирует смену инструмента на станке ЧПУ**: оно следит за цветом заданного пикселя на экране и при появлении нужного цвета автоматически нажимает на указанную точку интерфейса станка.

### Сценарий использования
- **Оператор** задаёт:
  - координаты пикселя, цвет которого нужно отслеживать (зона индикации необходимости смены инструмента);
  - координаты точки, куда нужно кликнуть (кнопка подтверждения / запуска процедуры на ЧПУ).
- При смене состояния станок **подсвечивает эту точку заданным цветом**.
- Программа видит изменение цвета и **однократно имитирует (двойной) клик мышью** по целевой точке, избавляя оператора от ручных действий.

### Архитектура и логика работы
- **Графический интерфейс (`Swing`)**
  - Главное окно `JFrame` с меткой текущих координат мыши (удобно наводиться на нужные точки).
  - Поля ввода координат:
    - `x`, `y` для точки проверки цвета;
    - `x`, `y` для точки клика.
  - Кнопки `start` / `stop` с цветовой индикацией активного состояния.
  - Окно фиксируется поверх остальных (`setAlwaysOnTop(true)`), чтобы не терять доступ к управлению.

- **Основной цикл (`update`)**
  - Каждые ~150 мс `Swing Timer` вызывает метод `update()`.
  - Метод:
    - обновляет отображаемые координаты текущего положения мыши;
    - считывает координаты проверки и координаты клика из полей ввода;
    - берёт цвет пикселя в точке проверки через `Robot.getPixelColor(x, y)`.

- **Условия срабатывания**
  - **Старт/стоп работы** контролируются флагом `running`, который переключают кнопки.
  - Если цвет проверяемого пикселя:
    - **белый (255,255,255)** или **тёмно‑красный (160,0,0)** – программа **автоматически останавливает работу**, чтобы не кликать в неправильный момент.
    - **бирюзовый (0,150,150)** – считается сигналом от ЧПУ о необходимости действия:
      - если клик ещё не выполнялся (флаг `clicked == false`), программа:
        - берёт координаты клика;
        - перемещает курсор и выполняет **двойной левый клик** через `Robot`;
        - выставляет `clicked = true`, чтобы не кликать повторно, пока цвет не изменится.
      - при любом другом цвете флаг `clicked` сбрасывается в `false`, готовя систему к следующему срабатыванию.

### Техническая часть
- **Стек**: Java SE, `Swing` (`JFrame`, `JPanel`, `JLabel`, `JTextField`, `JButton`, `Timer`), `java.awt.Robot` для чтения пикселей и эмуляции мыши.
- **Точка входа**: `com.example.tools.Tools` (запуск GUI через `SwingUtilities.invokeLater`).
- **Сборка и запуск**:
  - Проект собирается в JAR с манифестом (`META-INF/MANIFEST.MF`) и главным классом `com.example.tools.Tools`.
  - Запуск возможен через IDE (IntelliJ IDEA) или командой `java -jar <имя_собранного_jar>.jar` при установленной JRE/JDK.

В итоге приложение выступает как **простая и надёжная прослойка между визуальным интерфейсом станка ЧПУ и действиями оператора**, автоматизируя рутинные клики при смене инструмента и снижая риск человеческой ошибки.
